5. ▶ Play 链路 — 我**猜**的 3 个可能卡点（不修，先列出来）

卡点 A: _frames.Count == 0 静默早 return
ReplayTimeline.cs:148: if (_frames.Count == 0) return;
如果 LoadAsync 没成功 parse（_frames 是空），Play 静默 return，timer 不启动
检查方法: 你的 ASC 文件 _frames.Count 是几？LoadAsync 抛没抛异常？
卡点 B: OnTick 内 !_isPlaying 早 return
ReplayTimeline.cs:233: if (!_isPlaying) return;
如果 timer 启动后 _isPlaying 被某处改成 false，OnTick 一直早 return
检查方法: OnTick 第一次跑时 _isPlaying 是什么？
卡点 C: OnTick 跑了但 _emit 没调
ReplayTimeline.cs:264-273 while 循环需要 _frames[next].Timestamp <= now
如果 now 永远 ≤ 第一个 frame 的 Timestamp，循环不进入
检查方法: 第一个 frame 的 Timestamp 是几？PlayedTimestamp 算出来是多少？
