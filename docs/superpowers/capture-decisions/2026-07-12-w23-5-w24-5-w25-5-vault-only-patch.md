## Why consolidate into 1 cycle (vs 3 separate PATCH cycles)

User asked: "W23.5 + W24.5 + W25.5 都补一下，来得及吗?". Three separate PATCH cycles would each require:
- 1 branch creation + commit + PR + squash + tag + GH release (≈ 5 commands each)
- 1 post-PR docs commit (capture-decisions)
- 1+ vault captures (devlog + MEMORY)
- 3 CI runs (each ~2-3 min) = 6-9 min
- 3 version bumps forward (v3.39.5 + v3.39.6 + v3.39.7)

Single consolidated cycle (W17 sister precedent) = 1× all of the above but covers all 3 PATCH promotions in one go:
- 1 branch + 1 commit + 1 PR + 1 squash + 1 tag + 1 release
- 1 capture-decisions (this file)
- 1 CI run (~2-3 min)
- 1 version bump (v3.39.5)

**Trade-off**: The 3 lesson-promotions land in 1 atomic commit instead of 3 historical commits. Cleaner history but loses the per-cycle granularity. W17 sister-precedent (also 1 atomic commit) confirms this is acceptable.
