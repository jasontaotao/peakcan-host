## Approach

### Item 1 ÔÇö Dispose clears stale state (mirror v1.6.8 PATCH semantics)

Current `DbcApi.Dispose()` (lines 244-249):
```csharp
public void Dispose()
{
    _dbcService.DbcLoaded -= OnDbcLoaded;
    _dbcService.LoadFailed -= OnLoadFailed;
    _signalValues.Clear();
}
```

Gap: `_lastLoadError = null;` and `_currentDocument = null;` are NOT cleared. Per v1.6.9 pre-ship review LOW finding (opaque reference: pre-existing v1.6.8 behavior). One-shot DbcApi lifetime makes this defensive rather than critical, but pattern consistency (clearing on every state-mutation path) is the right discipline.

**Fix** (1 line, 2 statements):
```csharp
public void Dispose()
{
    _dbcService.DbcLoaded -= OnDbcLoaded;
    _dbcService.LoadFailed -= OnLoadFailed;
    _signalValues.Clear();
    // v1.6.10 PATCH Item 1: clear stale state on Dispose (mirror
    // OnDbcLoaded's success-side clearing at line 228).
    Volatile.Write(ref _currentDocument, null);
    _lastLoadError = null;
}
```

### Item 2 ÔÇö `PathOptions` + config-driven path allowlist

**New file**: `src/PeakCan.Host.Core/Path/PathOptions.cs`
```csharp
namespace PeakCan.Host.Core.Path;

/// <summary>
/// v1.6.10 PATCH Item 2: opt-in extension of v1.6.4 PATCH's hardcoded
/// <c>[PathNormalizer.LocalAppDataPeakCanRoot]</c> allowlist. Binds from
/// <c>Path:AllowedRoots:[]</c> in <c>appsettings.json</c> via AppHostBuilder.
/// </summary>
/// <param name="AllowedRoots">
/// Case-insensitive root directories the user may load JSON files from.
/// Backed by <see cref="PathNormalizer.NormalizeRestricted"/>'s
/// <c>IReadOnlyCollection&lt;string&gt;</c> parameter. Empty list rejects
/// every path (security hardening); not configured Ôćĺ <see cref="Default"/>
/// (v1.6.4 PATCH back-compat).
/// </param>
internal sealed record PathOptions(IReadOnlyList<string> AllowedRoots)
{
    /// <summary>
    /// v1.6.4 PATCH back-compat default: only <c>%LOCALAPPDATA%\PeakCan.Host</c>.
    /// Used when <c>Path:AllowedRoots</c> is absent from <c>appsettings.json</c>.
    /// </summary>
    public static PathOptions Default { get; } =
        new([PathNormalizer.LocalAppDataPeakCanRoot]);
}
```

**DidDatabase / RoutineDatabase ctor changes**: each gets a 3rd ctor param `PathOptions options`. Existing 1-arg + 2-arg ctors delegate with `PathOptions.Default`.


`LoadUserFile` (lines 52-109) replaces the hardcoded `[PathNormalizer.LocalAppDataPeakCanRoot]` (twice ÔÇö with-logger + null-logger paths) with `_options.AllowedRoots`. Same change for `RoutineDatabase.LoadUserFile` (lines 39-90).

New private field on each class: `private readonly PathOptions _options;`.

**AppHostBuilder DI** (mirror DbcOptions at lines 183-189):

## Decisions

### D1 ÔÇö Item 2 scope = extend existing restriction, NOT harden DbcService/Replay

Carry-over spec says "Config-driven path allowlist" (extension of v1.6.4 PATCH). Hardening DbcService path (currently uses `Normalize` not `NormalizeRestricted`) would re-trigger v1.6.4 PATCH 8th sub-shape of phase-2-5-brief-drift-correction: 5 test methods in `DbcServiceTests.cs` + `DbcServiceLimitTests.cs` use `%TEMP%` and would need migration to `%LOCALAPPDATA%\PeakCan.Host\`. Out of scope for v1.6.10 Tidy.

### D2 ÔÇö `PathOptions` in Core layer (mirror PathNormalizer location)

`PeakCan.Host.Core.Path.PathOptions` ÔÇö sits next to `PathNormalizer`. Core types (`DidDatabase`, `RoutineDatabase`) consume it directly. App layer reads config and constructs `PathOptions`. No NetArchTest rule 2 violation.

### D3 ÔÇö Default behavior: back-compat via `PathOptions.Default`

If `Path:AllowedRoots` is not configured Ôćĺ use `[LocalAppDataPeakCanRoot]` (v1.6.4 PATCH behavior). User can override via config.

If `Path:AllowedRoots:[]` is explicitly configured as empty array Ôćĺ reject all paths (security hardening per `PathNormalizer` convention: empty allowlist = reject everything).

If `Path:AllowedRoots` is non-empty Ôćĺ use the configured list.

### D4 ÔÇö DidDatabase / RoutineDatabase: add 3rd ctor param, preserve existing signatures

Mirrors `DbcService` 1-arg Ôćĺ 2-arg ctor delegation pattern. Back-compat:
- Existing 1-arg ctor (logger only) Ôćĺ delegates with `PathOptions.Default`
- Existing 2-arg ctor (path + logger) Ôćĺ delegates with `PathOptions.Default`
- NEW 3-arg ctor (path + logger + options) Ôćĺ primary

### D5 ÔÇö Item 1 = 2-line Dispose fix (defensive, mirror v1.6.8 OnDbcLoaded clearing)

Add `Volatile.Write(ref _currentDocument, null); _lastLoadError = null;` to `DbcApi.Dispose()`. Mirrors the clearing already done in `OnDbcLoaded` (v1.6.8 PATCH line 228).
