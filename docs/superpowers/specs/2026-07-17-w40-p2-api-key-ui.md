>
> 触发条件：v3.53.1 P1a (ICredentialStore + Windows Credential Manager) shipped 后，UI 输入界面被推迟到 v3.54.0/v3.55.0/v3.56.0/v3.57.0；4 个 release notes 均显式列为「未来 PATCH」或「P2 PATCH」。本 PATCH 完成 P1a 推迟项。

## 目标

为 DeepSeek LLM Provider 添加 WPF 原生 API Key 设置 UI：让用户能在 AI Analysis 面板内直接设置 / 验证 / 清除 API Key，无需通过 Windows Credential Manager GUI 或 `cmdkey` 命令行。技术债务兑现 v3.53.1 P1a 推迟的 UI 层。

## 5 个 D 决策（D1-D5）

| ID | 决策 | 选择 | 理由 |
|---|---|---|---|
| D1 | UI 入口位置 | AI Analysis 面板顶部**永久可见**的折叠区 + 状态指示器 | 用户进 AI Analysis tab 立刻看到 key 状态；不像临时 dialog 那样容易忘记 |
| D2 | API Key 输入控件 | **PasswordBox**（WPF 原生）+ 「显示/隐藏明文」切换 checkbox | 避免明文泄漏到 UI 自动化 / 屏幕截图；切换选项给 ops 调试用 |
| D3 | 状态指示器 | 3 态：`NotSet` / `Configured`（不显示 key 明文）/ `Removed` + 详细错误提示（provider 调用失败时） | 用户能立刻看到当前状态；不需要反复试 |
| D4 | API Key 持久化路径 | **复用现有 ICredentialStore**（不动 WindowsCredentialManagerStore） | v3.53.1 P1a 推迟承诺；不引入第二存储路径 |
| D5 | 「测试连接」按钮 | 包含；调用 DeepSeekProvider.AnalyzeAsync 的 stub 请求验证 HTTP 401 vs 200 | 减少用户反复 set+run 试错的成本 |

## 数据契约

### ApiKeyConfiguredState (new enum)
```csharp
public enum ApiKeyConfiguredState
{
    NotSet,       // GetAsync 返回 null
    Configured,   // GetAsync 返回非空
}
```

### ApiKeyStatus (new record)
```csharp
public sealed record ApiKeyStatus(
    ApiKeyConfiguredState State,
    string? LastError = null,
    DateTimeOffset? LastUpdatedAt = null);
```

### ApiKeyManager (new helper, ~80 LoC)
```csharp
public sealed class ApiKeyManager
{
    public const string CredentialKey = "deepseek-api-key";

    private readonly ICredentialStore _store;
    private readonly ILogger<ApiKeyManager> _logger;

    public ApiKeyManager(ICredentialStore store, ILogger<ApiKeyManager> logger) { ... }

    /// <summary>查询 key 是否已配置 + last error + last update time.</summary>
    public async Task<ApiKeyStatus> CheckAsync(CancellationToken ct = default);

    /// <summary>写入新 key. Always overwrites.</summary>
    public async Task SetAsync(string value, CancellationToken ct = default);

    /// <summary>删除 key. No-op if not set.</summary>
    public async Task RemoveAsync(CancellationToken ct = default);
}
```

### 关键不变量
- **NEVER 返回 key 明文**给调用方（API surface 只有 CheckAsync 返回 state，不返回 value）
- **NEVER log key 明文**（set 时 logger.LogInformation("API key updated"); 不 log value）

## 错误处理

- `ICredentialStore` 抛 `CredentialStoreException` 时：catch → `ApiKeyStatus.LastError = ex.Message` + log warning + 显示给用户
- PasswordBox 为空时：disable SetApiKeyCommand（CanExecute = !string.IsNullOrWhiteSpace）
- 已有 key 时：默认 PasswordBox 为空 + 提示「输入新 key 覆盖」

## 复评/不做项

- **WPF UI 自动化测试**: 不做（PasswordBox + dialog 复杂度不值得）
- **Streaming LLM response**: 不做（独立 PATCH 推迟项）
- **Azure OpenAI / Ollama 切换 UI**: 不做（独立 PATCH 推迟项）
- **DeepSeek 真实 API CI 集成测试**: 不做（独立 PATCH 推迟项）
- **Token usage 持久化**: 不做（独立 PATCH 推迟项）
- **Retry policy / exponential backoff**: 不做（用户手动 retry via UI）
- **API Key 历史审计**: 不做（single-key model per provider）
- **多 Provider 切换 UI**: 不做（独立 PATCH）
