## Per-Item Spec

### Item 1 — IsoTpLayer watchdog CTS race (HIGH)

**Symptom** (Phase 2.5 实地):
- `IsoTpLayer.cs:427` `StartReceiveWatchdog()` 入口先 `CancelReceiveWatchdog()` (line 429) 再创建新 CTS (line 463 old.Cancel())
- `_rxWatchdog` 是普通 nullable field (line 48, 非 volatile / Interlocked)
- 旧 CTS 的 `token.Register` 回调 (line 435-448) 持 `_rxLock` + 读 `_rxInProgress`
- `CancelReceiveWatchdog()` (line 461-465) 调 `old.Cancel()` → `old.Dispose()` 同步, 但 Register 回调可能仍在 `_rxLock` 上跑
- 触发条件: FF-then-CF 中途有 fresh FF (reassembly 还没完成) → `HandleConsecutiveFrameLocked` (line 537) 重启 watchdog → race

**Fix**:
1. `_rxWatchdog` 改为 opaque handle (object), 内部 own CTS + 一个 `Interlocked` `int _generation`; handle 比较时用 reference equality
2. `CancelReceiveWatchdog()` 改为: 创建新 handle, `Interlocked.Exchange(ref _rxWatchdog, newHandle)`, 然后 `old.Dispose()` 排到 `ThreadPool.QueueUserWorkItem` (`Dispose()` 在 Register 回调完成后执行)
3. `StartReceiveWatchdog` 入口接受 `expectedGeneration`, 仅在 `_rxInProgress` + generation 匹配时启动
4. 加 `internal long _watchdogDisposalDeferredCount` test hook

**Tests** (red 1, green 1):
- `tests/.../IsoTpLayerTests.cs`: 加 `HandleConsecutiveFrame_During_Watchdog_Disposal_Does_Not_Corrupt_Buffer` — 并发触发 ≥3 次 FF→CF burst + 在 Cancel 之前 await 短 delay, assert 无 watchdog 误 cancel 且 buffer 完整

**Files**: `src/PeakCan.Host.Core/Uds/IsoTp/IsoTpLayer.cs:427-470`

### Item 3 — UdsSession S3 keepalive OCE / Exception 合并 (MEDIUM)

**Symptom** (Phase 2.5 实地修正 brief):
- `UdsSession.cs:94-117` `StartS3KeepAlive` 回调
- `catch (OperationCanceledException)` (line 100) — brief 说 unreachable, 错. `Timer.Dispose()` (line 132) 不 await in-flight callback; shutdown 期间 callback 跑时 disposed CTS 抛 OCE, branch reachable
- 实际行为: OCE branch 与 `catch (Exception ex)` 行为几乎相同 (都 log), 唯一差异 OCE 不 increment `_s3Failures`
- Dead-code 陷阱: 看起来"special case"实际无作用

**Fix**:
1. `UdsSession.cs:100-117` 合并两个 catch arm:
   ```csharp
   catch (OperationCanceledException)
   {
       // Shutdown path: log but do not count as business failure
       LogS3KeepAliveShutdown(_logger);
   }
   catch (Exception ex)
   {
       LogS3KeepAliveFailed(_logger, ex);
       Interlocked.Increment(ref _s3Failures);
   }
   ```
2. 加 `[LoggerMessage(EventId = 2003, Level = LogLevel.Debug)]` `LogS3KeepAliveShutdown(ILogger? logger)` (OCE 路径用 Debug, 不污染 Warning 通道)
3. 不改 timer lifetime (保持 `Timer.Dispose()`); OCE 路径在 shutdown 仍 log 但不阻塞

**Tests**:
- `tests/.../UdsSessionTests.cs`: 加 `S3_KeepAlive_OperationCanceled_During_Shutdown_Logs_At_Debug` — mock ILogger, trigger Dispose-during-in-flight, assert Debug 级 log, 不增 _s3Failures
- 现有 `S3_KeepAlive_Does_Not_Swallow_Silently` (line 133, 147) 仍 pass (Exception 路径不变)

**Files**: `src/PeakCan.Host.Core/Uds/UdsSession.cs:94-117`

### Item 4 — UdsClient P2 timeout cancel _responseTcs.Task (HIGH)

**Symptom**:
- `UdsClient.cs:425` `_responseCts = CreateLinkedTokenSource(ct)`
- Line 428 `_responseCts.CancelAfter(_timer.P2Timeout)` — 触发 linked CTS 的 cancel
- Line 436 `await _responseTcs.Task` — 等待响应
- **问题**: `_responseCts.CancelAfter` 只 cancel linked CTS token, 不主动 set `_responseTcs`. 除非 caller await `_responseCts.Token` (没人), P2 timeout 不 unblock `await _responseTcs.Task` → 永久 hang
- **Phase 2.5 新发现**: `_responseCts.Dispose()` (line 451 finally) 之后, `OnMessageReceived` (line 464-477) 仍 `cts?.CancelAfter` 可能 OCE
- **Caller flow**: caller `await _responseTcs.Task` 卡死直到 OnMessageReceived 调 TrySetResult/Exception. P2 timeout 路径永不触发.

**Fix**:
1. `_responseCts.CancelAfter` 之前注册 callback:
   ```csharp
   var registration = _responseCts.Token.Register(
       static state => ((UdsClient)state!).OnP2TimeoutFired(),
       this);
   ```
2. `OnP2TimeoutFired()` 方法:
   ```csharp
   private void OnP2TimeoutFired()
   {
       var tcs = Volatile.Read(ref _responseTcs);
       tcs?.TrySetCanceled();
       // P2 timeout: P2* may still apply — but per ISO 14229, after P2 without response
       // the diagnostic session is dead; don't extend.
   }
   ```
3. `SendRequestInternalAsync` finally 顺序:
   ```csharp
   finally
   {
       registration.Dispose();           // 先 dispose registration
       Volatile.Write(ref _responseTcs, null);
       Volatile.Write(ref _responseCts, null);
       // existing _responseCts?.Dispose() — but only if not already
   }
   ```
4. `OnMessageReceived` (line 464-477) 加 `cts != null` guard (避免 disposed CTS 上 CancelAfter):
   ```csharp
   var cts = Volatile.Read(ref _responseCts);
   if (cts is null || cts.IsCancellationRequested) return;
   try { cts.CancelAfter(_timer.P2StarTimeout); }
   catch (ObjectDisposedException) { /* shutdown race, safe to ignore */ }
   ```
5. 加 `internal Action? OnP2TimeoutFiredForTesting { get; set; }` test hook

**Tests**:
- `tests/.../UdsClientTests.cs`: 加 `SendRequestAsync_P2Timeout_TriesSetCanceled_ResponseTcs` — 注入 long P2 timer, mock ICanBusClient 不回响应, assert OnP2TimeoutFiredForTesting 触发 + `await SendRequestAsync` throws `UdsException` (not TaskCanceledException)
- 加 `OnMessageReceived_After_Dispose_DoesNot_Throw` — dispose CTS 后调用 OnMessageReceived, assert 不抛

**Files**:
- `src/PeakCan.Host.Core/Uds/UdsClient.cs:412-490` (核心 SendRequestInternalAsync + OnMessageReceived)

### Item 7 — SendFrameLibrary.Count O(1) (LOW)

**Symptom**:
- `SendFrameLibrary.cs:156-165` `public int Count { get { lock (_gate) { return LoadUnlocked().Count; } } }`
- `LoadUnlocked` (line 167-181) 每次读整个 JSON file → O(file size)
- `SendViewModel.cs:341` 每次 save 后调 `_libraryService.Count` (status message) → UI thread 上额外 JSON parse

3. `Count` getter:
   ```csharp
   public int Count
   {
       get { lock (_gate) { EnsureLoaded(); return _cachedCount; } }
   }
   ```
4. `Add` (line 126) 在 `_gate` 内 `EnsureLoaded(); ...; _cachedCount++;`
5. `Remove` (line 141) 在 `_gate` 内 `EnsureLoaded(); ...; if (removed) _cachedCount--;`
6. `Save()` parameterless (line 108-115) 在 `_gate` 内 `EnsureLoaded(); SaveUnlocked(Frames);` (不重算 count)
