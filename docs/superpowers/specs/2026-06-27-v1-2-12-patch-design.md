## Decisions made in brainstorming

| # | Question | Decision | Why |
|---|----------|----------|-----|
| D1 | 14 项是一次 ship 还是分批 | **一次 ship, 单 squash-merge PR** | v1.2.11 PATCH review 显示分批会导致 v1.2.10 PATCH review 的 fix (SendFrameLibrary `_gate`) 在 v1.2.11 落地时又出回归; 一次 ship 锁住所有耦合修复 |
| D2 | CyclicSendService `OnTimerTick` 锁方案 | `lock(this)` snapshot at top of method | 与 `Start` / `StopInner` 用的同一把锁; 改动小, 不引入新 mutex |
| D3 | CyclicSendService count 拆分 | 拆 `_sendSuccessCount` / `_sendFailureCount` 两个 `Interlocked` | UI 现在只显示 success; 失败走 log throttle; 总数 = success + failure 单独绑到另一个 property |
| D4 | RecordService 重构策略 | **不重写**, 加 `Channel<CanFrame>` 缓冲 + 单独 writer thread; OnFrame 改为 `TryWrite` 非阻塞 | 最小 diff, 不动 `WriteHeader` / `WriteFooter` / `WriteFrame` 既有逻辑; 8 kfps back-pressure 由 Channel 容量限制 |
| D5 | RecordService 1Hz flush 实现 | 用 `PeriodicTimer` (1 Hz) 在 writer thread 中调 `_writer.Flush()` | .NET 8+ 原生支持, 不引入第三方 timer; 退出时由 `Dispose` 链取消 |
| D6 | VM `IDisposable` wire up 方案 | `AddSingleton<T>()` + `AddHostedService(sp => sp.GetRequiredService<T>())` 双注册 | 与 `TraceService` / `StatisticsService` / `DbcDecodeBackgroundService` 既有 pattern 一致; host `StopAsync` 自动触发 Dispose |
| D7 | App.OnStartup 改 async void vs ConfigureAwait 包装 | **改 `async void`**, 把整个 `OnStartup` 异步化 | WPF `Application.OnStartup` 推荐签名 (`protected override void OnStartup`); 改 `async void` 是 WPF 通行做法 (与 `ScriptView.OnLoaded` 同); 不留 SyncContext 风险 |
| D8 | ScriptView WebView2 失败 fallback UI | fallback `<TextBlock>` 显示 "WebView2 runtime 未安装" + 链接 | 不让一个 tab 加载失败杀整个 app; 用户仍能切到其他 tab; fallback 不阻塞脚本逻辑 |
| D9 | IsoTpLayer FF length 拒绝策略 | 在 `IsoTpFrame.DecodeFirstFrame` 入口 throw `ArgumentException`, `HandleFirstFrame` 入口直接 return 不抛 | 入口拒绝避免 state 污染; 抛异常路径与现有 watchdog cancel 路径对齐 |
| D10 | ChannelRouter 注入 `ILogger` vs 继续用 `Debug.WriteLine` | **注入** `ILogger<ChannelRouter>`, nullable ctor 参数 (测试可传 null) | 跟随 D-ME-5 / D-LO-3 的 sink logger 模式; 测试可构造 null-safe |
| D11 | `IsOk` 改 bitwise 评估 vs 显式 strip 后 switch | 显式 strip 后 switch (新方法 `GetBaseError(uint raw)`); `IsOk` 改用 `(raw & ~flagMask) == OK` | 保留原 `IsOk` 调用方; `ToErrorCode` 走 strip 后 switch, composite code 落到具体 case |
| D12 | SendFrameLibrary `File.Replace` vs `File.Move` | `File.Replace(sourceFileName, destinationFileName, destinationBackupFileName: null)` | Win10 1809+ 等价 POSIX rename; 保留 ACL/attributes; backup null = 不留 .bak |
| D13 | SendFrameLibrary 显式 `Encoding.UTF8` | 是 | v1.2.12 起 JSON 持久化文件名可能是中文; 不带 BOM 在 Notepad 打开乱码; 带 BOM 是 Windows 用户预期 |
| D14 | UdsClient `_responseTcs` volatile 范围 | `Volatile.Read` / `Volatile.Write` 包所有读写; 同步 `OnMessageReceived` 入口 | 最小改动; 不重构成 lock-free; 与既有 `_pendingRequestSid` 比对顺序不变 |
| D15 | 错误日志级别 | sink `OnError` 用 `Warning` (transient) + `Critical` (data loss); `UdsSession` S3 keepalive 用 `Warning` | 与既有 pattern 一致 (如 `LogFrameWriteFailed` 是 Warning) |
| D16 | Commit 策略 | **每 Task 一个 commit**, TDD 流程 (RED → GREEN → IMPROVE), 末尾 ship prep 单独 commit | 与 v1.2.10 / v1.2.11 PATCH 一致; squash-merge 时 squash 为 ~12-16 commits |

## Per-Item Spec

### Item 1 — SendFrameLibrary regression (HIGH)

**Symptom**: v1.2.11 PATCH 在 `SendFrameLibrary.cs:103-129` 加了 `Add(SavedFrame)` / `Remove(string)` 原子方法 (用 `_gate` lock), 但 `SendViewModel.cs:310-312` 的 `SaveCurrentToLibrary` 和 `:334-336` 的 `DeleteFromLibrary` 仍然走非原子的 `Load → ToList → Add/RemoveAll → Save` 路径。并发调用 (快速点击 Save / Delete 按钮) 时丢修改, 后调用的 Save 覆盖前调用的修改。**这是 v1.2.11 修复要关掉的 race, 回归了。**

**Fix**:
1. `SendViewModel.SaveCurrentToLibrary` (line 307-315) 改为:
   ```csharp
   var saved = ToSavedFrame();
   try
   {
       _libraryService.Add(saved);
       RefreshLibrary();
       Status = $"Saved '{saved.Name}' to library ({_libraryService.Count} frames).";
   }
   catch (Exception ex)
   {
       Status = $"FAIL: Save '{saved.Name}' to library: {ex.Message}";
       LogSaveToLibraryFailed(_logger, ex, saved.Name);
   }
   ```
2. `SendViewModel.DeleteFromLibrary` (line 331-339) 改为:
   ```csharp
   if (frame is null) return;
   try
   {
       if (_libraryService.Remove(frame.Name))
       {
           RefreshLibrary();
           Status = $"Removed '{frame.Name}' from library.";
       }
       else
       {
           Status = $"'{frame.Name}' not found in library (already removed?).";
       }
   }
   catch (Exception ex)
   {
       Status = $"FAIL: Remove '{frame.Name}': {ex.Message}";
       LogDeleteFromLibraryFailed(_logger, ex, frame.Name);
   }
   ```
3. 加 `[LoggerMessage]` source-gen helpers:
   ```csharp
   [LoggerMessage(EventId = 2001, Level = LogLevel.Error, Message = "Save '{Name}' to library failed")]
   private static partial void LogSaveToLibraryFailed(ILogger logger, Exception ex, string name);
   [LoggerMessage(EventId = 2002, Level = LogLevel.Error, Message = "Delete '{Name}' from library failed")]
   private static partial void LogDeleteFromLibraryFailed(ILogger logger, Exception ex, string name);
   ```
4. `SendFrameLibrary` 暴露 `public int Count { get { lock (_gate) return LoadUnlocked().Count; } }` 用于 status message.

**Behavior matrix**:

| 场景 | 旧 (v1.2.11) | 新 (v1.2.12) |
|------|--------------|--------------|
| 单次 Save | OK | OK, 加 success status |
| 单次 Delete | OK | OK, 加 success status |
| 并发 Save A + Save B (双击) | A's Save 覆盖 B's Save | A 和 B 都 add, 列表中两条 |
| 并发 Save A + Delete B (混点) | Save 覆盖 Delete | A add 成功, B remove 成功, 列表中 A |
| Save 失败 (磁盘满 / AV 锁) | `DispatcherUnhandledException` 杀进程 | Status 显示 "FAIL: ..." 走 `Start/Stop` 同模式 |
| 删不存在的项 | 静默 no-op | Status 显示 "not found" |

**Tests**:
- `SendFrameLibraryTests.Add_Then_List_Contains_Added`
- `SendFrameLibraryTests.Add_Twice_Distinct_Names_Both_Kept`
- `SendFrameLibraryTests.Add_Twice_Same_Name_Replaces`
- `SendFrameLibraryTests.Remove_Existing_Returns_True`
- `SendFrameLibraryTests.Remove_NonExistent_Returns_False`
- `SendFrameLibraryTests.Concurrent_Add_From_Two_Threads_Both_Kept` (竞争回归测试)
- `SendViewModelTests.SaveCurrentToLibrary_Adds_Via_Atomic_Method` (验证调用 `Add` 而非 `Load+Save`)

### Item 2 — IsoTpLayer callback async Task (HIGH, 1.2)

**Symptom**: `AppHostBuilder.cs:161-174` 在 `IsoTpLayer` 工厂里用 `sendService.SendAsync(frame).AsTask().Wait()` 同步等待 async send。无 timeout, 无 try/catch。在 SDK 读线程 (IsoTpLayer 在哪条线程上 fire callback?) 上阻塞; `AggregateException` 被 `.Wait()` 吞掉。ISO-TP 通道永久挂起时整个 UDS 诊断面死锁。

**Fix**:
1. 改 `IsoTpLayer` ctor 签名: `Func<CanFrame, Task>?` 替代 `Action<CanFrame>?` (nullable 兼容)
2. `IsoTpLayer.SendMultiFrameAsync` (line 434-444) 把 `_sendFrame` 调用点改为 `await _sendFrame(frame).ConfigureAwait(false)`, 用 try/catch 包裹
3. `AppHostBuilder.cs:172` 改为:
   ```csharp
   return new IsoTpLayer(config, async frame =>
   {
       try { await sendService.SendAsync(frame).ConfigureAwait(false); }
       catch (Exception ex) { LogIsoTpSendFailed(_logger, ex, frame.Id.Raw); }
   });
   ```
4. ISO-TP 串行化: 用 `SemaphoreSlim(1, 1)` 包 send path, 保证 FF/CF 顺序
5. 加 `[LoggerMessage]`:
   ```csharp
   [LoggerMessage(EventId = 3001, Level = LogLevel.Error, Message = "IsoTpLayer send failed for ID 0x{Id:X}")]
   private static partial void LogIsoTpSendFailed(ILogger logger, Exception ex, uint id);
   ```
6. `AppHostBuilder` 注入 `ILogger<>` 即可 (现成的 `Serilog` 配置)

**Behavior matrix**:

| 场景 | 旧 | 新 |
|------|----|----|
| 正常 send | OK (但阻塞 SDK 线程) | OK, 异步, 不阻塞 |
| SDK 故障永久挂起 | 死锁, 整个 UDS 通道死 | 日志记一次, ISO-TP 继续接受下一帧 (timeout 由上层 P2* 兜底) |
| 并发 FF/CF 发送 | 无序风险 (与 ISO 15765-2 §6.5 冲突) | 串行化, 顺序保证 |
| 异常抛出 | `AggregateException` 吞 | log Error, 走 watchdog recovery |

### Item 3 — IsoTpLayer Monitor.Exit/Enter try/catch (HIGH, B-CR-1)

**Symptom**: `IsoTpLayer.cs:389-398` 在 `_rxLock` 内修改完 `_rxInProgress` / `_rxBuffer` 后, `Monitor.Exit(_rxLock)` → `try { handler?.Invoke(complete); } finally { Monitor.Enter(_rxLock); }`. **无 `catch`**. Subscriber (`UdsClient.OnMessageReceived`) 抛异常时, exception 传播**且** lock 状态破坏 (`_rxInProgress=false` / `_rxBuffer=null` 已被 setting, 但 `Monitor.Enter` 还在 finally 中重入). 接收状态机进入不一致状态.

**Fix**:
1. 在 `IsoTpLayer.HandleConsecutiveFrame` 拆成两个方法:
   - `HandleConsecutiveFrameLocked` (lock 内): sequence 校验 + 数据 copy + 返回 `ReassembledMessage?`
   - `HandleConsecutiveFrameUnlocked` (lock 外): `try { handler?.Invoke(complete); } catch (Exception ex) { LogIsoTpHandlerFailed(_logger, ex, complete.Length); }`
2. 调用方在 lock 外 invoke `handler`, 不做 `Monitor.Exit` / `Monitor.Enter` 反模式
3. 加 `[LoggerMessage]`:
   ```csharp
   [LoggerMessage(EventId = 3002, Level = LogLevel.Error, Message = "IsoTpLayer MessageReceived handler threw for {Length}-byte message")]
   private static partial void LogIsoTpHandlerFailed(ILogger logger, Exception ex, int length);
   ```
4. `IsoTpLayer` 注入 `ILogger<IsoTpLayer>` (nullable for tests)

**Behavior matrix**:

| 场景 | 旧 | 新 |
|------|----|----|
| Subscriber 正常 | OK | OK |
| Subscriber 抛 | exception 传播, lock state 破坏, ISO-TP 接收挂死 | exception log + ISO-TP 状态保持干净 (lock 持有者 exit 后正常) |
| 连续多帧, 中间 subscriber 抛 | 后续帧可能错配 | 后续帧正常处理 |

**Tests**:
- `IsoTpLayerTests.MessageReceived_Handler_Throws_Does_Not_Corrupt_State`
- `IsoTpLayerTests.MessageReceived_Handler_Throws_Is_Logged_And_Not_Propagated`
- `IsoTpLayerTests.MessageReceived_Next_Frame_After_Handler_Throw_Works`

### Item 4 — App.OnStartup async void (HIGH, B-HI-1)

**Symptom**: `App.xaml.cs:95` `_host.StartAsync().GetAwaiter().GetResult()` 在 WPF STA UI 线程上同步等待. 当前 1 Hz BackgroundService 是 threadpool-based, 不会 deadlock, 但 `OnExit` 也有同样问题 (`StopAsync(10s)` 是有 timeout, 略安全). 一旦未来 hosted service 在 `StartAsync` 内部 `await` 后 capture STA SyncContext, 即死锁. **不对称**: `StartAsync` 无 timeout, `StopAsync` 有 10s timeout (line 123).

**Fix**:
1. `App.OnStartup` 改 `async void`:
   ```csharp
   protected override async void OnStartup(StartupEventArgs e)
   {
       base.OnStartup(e);
       try
       {
           await _host.StartAsync().ConfigureAwait(false);  // 不 capture STA
       }
       catch (Exception ex)
       {
           Log.Logger.Fatal(ex, "Host start failed");
           Shutdown(1);
           return;
       }
       _shell.Show();
   }
   ```
2. `OnExit` 也改 `async void`, 但保留 10s `StopAsync` timeout + bounded wait:
   ```csharp
   protected override async void OnExit(ExitEventArgs e)
   {
       base.OnExit(e);
       if (_host is null) return;
       try
       {
           await _host.StopAsync(TimeSpan.FromSeconds(10)).ConfigureAwait(false);
       }
       catch (Exception ex)
       {
           Log.Logger.Error(ex, "Host stop failed; continuing exit");
       }
       finally
       {
           _host.Dispose();
           _host = null;
       }
   }
   ```
3. `OnStartup` 中 `_shell.Show()` 移到 `await _host.StartAsync()` 之后, 保证 host 完全启动才显示 UI

**Behavior matrix**:

| 场景 | 旧 | 新 |
|------|----|----|
| 正常启动 | OK | OK |
| Hosted service StartAsync 抛 | OK (catch + log + throw) | OK (catch + Shutdown(1) 而不是 throw) |
| 启动时 deadlock | 理论 (未来 hosted service 加) | 无 (async) |
| OnExit 10s 限制 | 有 | 保留 |

**Tests**:
- 不写新 xunit 测试 (WPF App 是 STA 不可单测); 靠现有 `AppLifecycleTests` 覆盖 (实测集成测试)
- 更新 `AppLifecycleTests` 验证 host 启动失败时 `Shutdown(1)` 而非 throw

### Item 5 — RecordService Channel + writer thread + 1Hz flush (HIGH, 1.4)

**Symptom**: 
- `RecordService.cs:134-153` `OnFrame` 在 SDK 读线程上 `lock (_gate) { WriteFrame(frame); _frameCount++; }` —— 持锁跨 StreamWriter write. 慢盘/网络盘/AV 锁会 back-pressure SDK 读线程, 丢 CAN 帧.
- Class doc (line 16-17) 承诺 "flushed periodically (every 1 s or on stop)" 但实现里**没有** `Timer` / `PeriodicTimer` / 任何周期 flush. 只有 `StopRecordingInner` 调 `_writer.Flush()`. Doc 撒谎.

**Fix**:
1. `RecordService` 加 `Channel<CanFrame>` 字段: `private readonly Channel<CanFrame> _frameChannel = Channel.CreateBounded<CanFrame>(new BoundedChannelOptions(8192) { FullMode = BoundedChannelFullMode.DropOldest, SingleReader = true, SingleWriter = true });`
2. `OnFrame` 改非阻塞:
   ```csharp
   public void OnFrame(CanFrame frame)
   {
       if (!_isRecording || _writer is null) return;
       _frameChannel.Writer.TryWrite(frame);  // 非阻塞, 满了丢最老
       Interlocked.Increment(ref _frameEnqueuedCount);
   }
   ```
3. `ExecuteAsync` (override `BackgroundService`) 启动 writer loop:
   ```csharp
   protected override async Task ExecuteAsync(CancellationToken stoppingToken)
   {
       var flushTimer = new PeriodicTimer(TimeSpan.FromSeconds(1));
       try
       {
           await foreach (var frame in _frameChannel.Reader.ReadAllAsync(stoppingToken))
           {
               try
               {
                   WriteFrame(frame);
                   Interlocked.Increment(ref _frameCount);
               }
               catch (Exception ex)
               {
                   LogFrameWriteFailed(_logger, ex);
               }
           }
       }
       catch (OperationCanceledException) { /* shutdown */ }
   }
   ```
