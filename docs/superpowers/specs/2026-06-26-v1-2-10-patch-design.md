## Item A — Stats chart 不 redraw

### Symptom (用户 2026-06-26 报告)

> "数据没在滚动，只有 0-60 的数据，后续数据没在变化了"

### Root cause

`StatsViewModel.Apply` (`src/PeakCan.Host.App/ViewModels/StatsViewModel.cs:204-233`) 每 1 Hz 重建 `_fpsLine.Points` 和 `_loadLine.Points`，但**末尾没有调用 `PlotModel.InvalidatePlot(...)`**。OxyPlot 2.2.0 WPF 不会监听 `LineSeries.Points` 列表的 mutation，必须显式 invalidate 才会重绘。

### 对照参照（正确实现）

`SignalChartViewModel.cs:325-356` `OnRenderTick` 末尾：
```csharp
// ... 写 series.Points ...
PlotModel.InvalidatePlot(true);   // ← 正确写法
```

`SignalChartViewModel.cs:161` `AddSignal`：
```csharp
PlotModel.InvalidatePlot(false);
```

### v1.2.9 PATCH 怎么漏的

回看 `docs/release-notes-v1.2.9.md` §Bug 2 Fix：
> Rebuild `_fpsLine.Points` / `_loadLine.Points` from the `ObservableCollection` on every `Apply`.

只描述了"重填 Points"，**没提 InvalidatePlot**。v1.2.9 整批 ship 前没人手动跑 + 看 chart 是否真的滚动。

### Fix

`StatsViewModel.Apply` 末尾加：
```csharp
PlotModel.InvalidatePlot(updateData: true);
```

`updateData: true` 让 OxyPlot 重算 `Points`（否则只 trigger redraw 不重算数据 layout）。

### 验证

- **单元测试**：订阅 `PlotModel.Updated` event，断言 `Apply` 后 event 被触发
- **手动验证**：连 PEAK CAN，跑起来，chart 应每 1 Hz 滚动；Total / Error 数字应随之更新

## Item B — Signal Click handler 竞态

### Symptom

> "plot 和 取消 plot 都不太好点，有时候多点了几次，显示没勾选，但是数据线在更新"
> "在无内容接收时，能正确的点击和取消" (用户 2026-06-26 补充)

### Root cause

`SignalView.xaml.cs:53-60` `OnPlotCheckboxClick`：
```csharp
private void OnPlotCheckboxClick(object sender, RoutedEventArgs e)
{
    if (sender is CheckBox { DataContext: SignalEntry entry }
        && DataContext is SignalViewModel vm)
    {
        vm.OnSignalSelectionChanged(entry.Message, entry.Signal, entry.IsSelected);  // ← 读 entry.IsSelected
    }
}
```

`entry = sender.DataContext` 是 row 的 SignalEntry。但 `SignalViewModel.Upsert` (`SignalViewModel.cs:506-523`) 在每帧（最高 8 kfps）会**替换** Latest[i]：
```csharp
entry.IsSelected = Latest[i].IsSelected;  // 拷贝 IsSelected 到新 entry
Latest[i] = entry;                         // 替换为新 entry
```

时序竞态（uncheck 已 plot 信号 + 数据流动中）：
1. t=0  用户点 CheckBox，意图 uncheck
2. t=0  WPF toggle CheckBox.IsChecked UI（true→false）
3. t=0  Binding push `unchecked` 到 **OLD entry**（当时还在 Latest）
4. t=1  DrainPending 跑：Upsert 创建 NEW entry，把 OLD entry.IsSelected=true 拷贝到 NEW
5. t=1  `Latest[i] = NEW entry`，DataGrid 回收 row，CheckBox binding 重新 target 到 NEW entry
6. t=2  Click handler 触发，读 `sender.DataContext` → NEW entry，NEW entry.IsSelected=true → **AddSignal**（而不是 RemoveSignal）
7. t=3  用户看到的 CheckBox 视觉是 unchecked（binding push 在 t=0 已成功），但 chart line 反而被 Add 了

无数据时没有 DrainPending 替换，`sender.DataContext` 始终是同一个 entry，handler 读到的值正确。

### Fix

把"读 entry.IsSelected"改成"读 sender.IsChecked"。CheckBox.IsChecked 是 UI 侧 DP，在 Click event 触发时**一定是用户刚 toggle 的新值**（WPF 的 Toggle → Click 顺序由 ButtonBase 决定，不受 binding 状态影响）。

具体改动：
1. `SignalViewModel` 加 public/internal 方法 `HandlePlotCheckboxClick(SignalEntry entry, bool isChecked)` 把决策路径下沉到 VM，便于单元测试
2. `SignalView.xaml.cs` `OnPlotCheckboxClick` 改读 `cb.IsChecked` 而不是 `entry.IsSelected`

```csharp
// SignalViewModel.cs 新增
public void HandlePlotCheckboxClick(SignalEntry entry, bool isChecked)
    => OnSignalSelectionChanged(entry.Message, entry.Signal, isChecked);

// SignalView.xaml.cs 修改
private void OnPlotCheckboxClick(object sender, RoutedEventArgs e)
{
    if (sender is CheckBox { IsChecked: bool isChecked } cb
        && cb.DataContext is SignalEntry entry
        && DataContext is SignalViewModel vm)
    {
        vm.HandlePlotCheckboxClick(entry, isChecked);
    }
}
```

### 验证

- **单元测试**：构造 NEW entry.IsSelected=true（旧值已 preserve）但 cb.IsChecked=false（用户刚 uncheck）的场景，断言调用 RemoveSignal 而非 AddSignal
- **手动验证**：连 PEAK CAN，DBC 加载，flow 数据，反复点 plot checkbox，确认 chart line 与 visual 严格同步

## Item C — Stats OxyPlot Legend

### Symptom (carry-over from v1.2.9 release notes §"Known issue")

> Stats tab `OxyPlot` `Legend` is empty by default in OxyPlot 2.2.0 (no legend rendering for the FPS / bus-load series labels).

### Root cause

`StatsViewModel` 构造时 `new PlotModel { ... }` 没注册 `Legend`。OxyPlot 2.2.0 默认不渲染 legend series 标签。

### Fix

构造 `PlotModel` 后加：
```csharp
PlotModel.Legends.Add(new Legend
{
    LegendPlacement = LegendPlacement.Outside,
    LegendPosition = LegendPosition.RightTop,
    LegendBackground = OxyColor.FromAColor(32, OxyColor.FromRgb(0xFF, 0xFF, 0xFF)),
    LegendBorder = OxyColor.FromRgb(0xCC, 0xCC, 0xCC),
});
```

### 验证

- **单元测试**：断言 `PlotModel.Legends.Count == 1`，且 `LegendPlacement == LegendPlacement.Outside`
- **手动验证**：打开 Stats tab，看右上角 "Frames / sec" + "Bus load %" 两个 series 标签

## Item D — StatsViewModelTests flake 修复

### Symptom (carry-over from v1.2.9 release notes)

> `Push_Trims_Series_To_MaxPoints_After_Many_Samples` failed once in full-suite (parallel-execution App leak, same root cause as v1.2.6 dispatcher flake)

### Root cause

xunit 并行执行时，STA Application leak 污染下一个测试的 Application.Current 状态。v1.2.1 PATCH 已修复同样模式的 flake 在 SignalViewModelTests / DbcDecodeBackgroundServiceTests / DbcDecodeBackgroundServiceTests / TraceViewModelTests，用 `LeakedApplicationReset.CleanupLeakedApplication()` 放在 ctor。

StatsViewModelTests 漏掉这个 cleanup。

### Fix

`tests/PeakCan.Host.App.Tests/ViewModels/StatsViewModelTests.cs` ctor 加 `LeakedApplicationReset.CleanupLeakedApplication();`，参照 v1.2.1 的 4 个测试类（SignalViewModelTests, StatsViewModelTests, TraceViewModelTests, DbcDecodeBackgroundServiceTests）。

### 验证

- 全量跑 5 次，记录 flake 次数；目标：5/5 通过
- 单独跑该测试 100 次，记录 flake 次数；目标：0/100 flake

### ⚠️ 发现（2026-06-26 写 plan 时核实）

读 `tests/PeakCan.Host.App.Tests/ViewModels/StatsViewModelTests.cs` 第 28-41 行确认：

```
<see cref="LeakedApplicationReset.CleanupLeakedApplication"/> ...

```

**v1.2.1 已经把这个 cleanup 加到 StatsViewModelTests ctor 了**。v1.2.9 release notes 的"StatsViewModelTests ctor needs LeakedApplicationReset"是 stale 记录（v1.2.9 ship 时漏看了 v1.2.1 已修）。Item D **无代码变更**，仅 release notes 中说明此 carry-over 已由 v1.2.1 解决。
