**Symptom (recap from spec §Item 1):** `SendMultiFrameAsync` finally (`IsoTpLayer.cs:432-435`) only releases `_sendGate`. After any throw (FF/CF `IsoTpSendFailedException`, FC `TimeoutException`, `OperationCanceledException`), `_txWaitingForFc` stays `true`. The next `SendMessageAsync` enters `WaitForFlowControlAsync` (line 459) → busy-waits on `Task.Delay(1, timeoutCts.Token)` until 1000 ms `FlowControlTimeout` fires → `TimeoutException("No Flow Control received")` (line 382) → channel silently wedges all subsequent multi-frame transports. **Latent before v1.2.13 Item 5; acute after.**

- [ ] **Step 1: Add RED test `SendMultiFrameAsync_After_SendFailure_Resets_TxWaitingForFc`**

```csharp
/// <summary>
/// v1.2.14 PATCH Task 1: regression test for the _txWaitingForFc leak
/// introduced by v1.2.13 PATCH Item 5 throw path. Without the finally
/// reset, the second SendMultiFrameAsync after a failing first send
/// blocks forever in WaitForFlowControlAsync (1000ms timeout) because
/// _txWaitingForFc was never cleared. With the fix, the second send
/// must succeed as soon as a real FC arrives.
/// </summary>
[Fact]
public async Task SendMultiFrameAsync_After_SendFailure_Resets_TxWaitingForFc()
{
    var sink = new ObservableCollection<byte[]>();
    var iso = NewLayer(sink);
    iso.FlowControlTimeout = TimeSpan.FromMilliseconds(200);

    // First send: CF2 will fail (count=2 frame fires the InjectCfAsync
    // delegate that throws). The exception propagates out of
    // SendMultiFrameAsync as IsoTpSendFailedException.
    var firstPayload = new byte[14]; // FF(20) - actually 14 fits in single FF for test,
                                     // use 14 to force multi-frame: 6 + 8 CFs
    for (int i = 0; i < firstPayload.Length; i++) firstPayload[i] = (byte)(i + 1);

    var firstAttempt = () => iso.SendMessageAsync(0x123, firstPayload, CancellationToken.None);
    await firstAttempt.Should().ThrowAsync<IsoTpSendFailedException>();

    // Second send immediately after the failure: with the bug, this
    // would block on _txWaitingForFc=true (residual from first attempt)
    // and time out after FlowControlTimeout. With the fix, _txWaitingForFc
    // is reset to false in finally, so this sends FF immediately.
    var secondPayload = new byte[] { 0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF };
    var secondAttempt = async () =>
    {
        using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(2));
        await iso.SendMessageAsync(0x456, secondPayload, cts.Token).ConfigureAwait(false);
    };

    // Inject a synthetic FC immediately so WaitForFlowControlAsync exits.
    // The key invariant: the second attempt must NOT block for the full
    // 200ms FlowControlTimeout because _txWaitingForFc must already be false.
    InjectFlowControl(iso, blockSize: 0, minSeparation: 0);

    // Should complete well under the 2-second cancellation budget.
    var sw = System.Diagnostics.Stopwatch.StartNew();
    await secondAttempt.Should().NotThrowAsync();
    sw.Stop();
    sw.ElapsedMilliseconds.Should().BeLessThan(500,
        "second send must not busy-wait on stale _txWaitingForFc=true");
}
```

```csharp
/// <summary>
/// v1.2.14 PATCH Task 3: mirror of v1.2.13 Item 1's CF→FF watchdog
/// disposal test, covering the FF→FF path. The production hardening at
/// IsoTpLayer.cs:554-586 (3-guard: _rxWatchdog is null + Generation !=
/// expected + _rxInProgress) was added in v1.2.13 but only one half of
/// the test pair shipped. This test closes that gap.
/// </summary>
[Fact]
public async Task HandleFirstFrame_During_Watchdog_Disposal_Does_Not_Corrupt_Buffer()
{
    var sink = new ObservableCollection<byte[]>();
    var iso = NewLayer(sink);
    iso.ReceiveTimeout = TimeSpan.FromMilliseconds(80);

    // FF1: totalLength=20, firstChunk=6 bytes [0x01..0x06]
    InjectFirstFrame(iso, totalLength: 20, firstChunk: new byte[] { 1, 2, 3, 4, 5, 6 });

    // Immediately inject FF2: totalLength=20, firstChunk=6 bytes [0x11..0x16].
    // This forces CancelReceiveWatchdog on FF1's CTS mid-disposal
    // (HandleFirstFrame line 538) + StartReceiveWatchdog for FF2.
    InjectFirstFrame(iso, totalLength: 20, firstChunk: new byte[] { 0x11, 0x12, 0x13, 0x14, 0x15, 0x16 });

    // Wait long enough for FF1's late timer callback to potentially fire
    // (~80 ms ReceiveTimeout + scheduler slack).
    await Task.Delay(150);

    // Now complete FF2's reassembly: send CF1 sequence=1 with the remaining 14 bytes.
    var cf1Data = new byte[14];
    for (int i = 0; i < cf1Data.Length; i++) cf1Data[i] = (byte)(0x20 + i);
    InjectConsecutiveFrame(iso, sequence: 1, data: cf1Data);

    // Assert: FF2 reassembled end-to-end with its full 20-byte payload.
    sink.Should().ContainSingle()
        .Which.Should().Equal(new byte[]
        {
            0x11, 0x12, 0x13, 0x14, 0x15, 0x16,
            0x20, 0x21, 0x22, 0x23, 0x24, 0x25,
            0x26, 0x27, 0x28, 0x29, 0x2A, 0x2B, 0x2C, 0x2D,
        });

    // Assert: the deferred-disposal path was exercised at least once.
    Assert.True(iso._watchdogDisposalDeferredCount >= 1,
        $"Expected _watchdogDisposalDeferredCount >= 1, got {iso._watchdogDisposalDeferredCount}");
}
```
