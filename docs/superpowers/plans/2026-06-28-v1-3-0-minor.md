- [ ] **Step 4: Modify `UdsSecurity.cs` — add lockout state**

Edit `src/PeakCan.Host.Core/Uds/UdsSecurity.cs`. Add the `AttemptCount` + `LockedUntilUtc` fields to the nested `SecurityLevelState` class, add `LockoutConfig` property, and add the `IsLocked` / `RemainingLockoutDelay` / `RecordFailedAttempt` / `ResetLockout` methods:

```csharp
namespace PeakCan.Host.Core.Uds;

/// <summary>
/// UDS security access state management. Tracks seed/key exchange,
/// authentication status, and lockout state per security level.
/// </summary>
public sealed class UdsSecurity
{
    private readonly Dictionary<byte, SecurityLevelState> _levels = new();

    /// <summary>
    /// v1.3.0 MINOR Item 1: lockout policy. Defaults to 3 attempts / 5 s.
    /// Settable via <c>UdsClient</c> DI or directly for tests.
    /// </summary>
    public UdsSecurityLockoutConfig LockoutConfig { get; set; } =
        UdsSecurityLockoutConfig.Default;

    /// <summary>Get the current seed for a security level.</summary>
    public byte[]? GetSeed(byte level)
    {
        lock (_levels)
        {
            return _levels.TryGetValue(level, out var state) ? state.Seed : null;
        }
    }

    /// <summary>Set the seed received from ECU.</summary>
    public void SetSeed(byte level, byte[] seed)
    {
        lock (_levels)
        {
            _levels[level] = new SecurityLevelState
            {
                Seed = seed,
                IsAuthenticated = false
            };
        }
    }

    /// <summary>Mark a security level as authenticated.</summary>
    public void SetAuthenticated(byte level)
    {
        lock (_levels)
        {
            if (_levels.TryGetValue(level, out var state))
            {
                state.IsAuthenticated = true;
            }
        }
    }

    /// <summary>Check if a security level is authenticated.</summary>
    public bool IsAuthenticated(byte level)
    {
        lock (_levels)
        {
            return _levels.TryGetValue(level, out var state) && state.IsAuthenticated;
        }
    }

    /// <summary>Clear authentication for all levels (e.g., on session change).</summary>
    public void Reset()
    {
        lock (_levels)
        {
            _levels.Clear();
        }
    }

    /// <summary>
    /// v1.3.0 MINOR Item 1: true if the security level is currently locked
    /// out (host-side enforcement). Lockout state is independent of session
    /// state — it is NOT cleared by <see cref="Reset"/> (D8 from spec).
    /// </summary>
    public bool IsLocked(byte level)
    {
        lock (_levels)
        {
            if (_levels.TryGetValue(level, out var state) && state.LockedUntilUtc is DateTime until)
                return DateTime.UtcNow < until;
            return false;
        }
    }

    /// <summary>
    /// v1.3.0 MINOR Item 1: remaining lockout time. Returns
    /// <see cref="TimeSpan.Zero"/> if not locked.
    /// </summary>
    public TimeSpan RemainingLockoutDelay(byte level)
    {
        lock (_levels)
        {
            if (_levels.TryGetValue(level, out var state) && state.LockedUntilUtc is DateTime until)
            {
                var remaining = until - DateTime.UtcNow;
                return remaining > TimeSpan.Zero ? remaining : TimeSpan.Zero;
            }
            return TimeSpan.Zero;
        }
    }

    /// <summary>
    /// v1.3.0 MINOR Item 1: record a failed authentication attempt.
    /// Increments counter; on reaching <see cref="UdsSecurityLockoutConfig.MaxAttempts"/>
    /// sets <see cref="SecurityLevelState.LockedUntilUtc"/> and resets counter.
    /// </summary>
    internal void RecordFailedAttempt(byte level)
    {
        lock (_levels)
        {
            var state = _levels.TryGetValue(level, out var s) ? s : new SecurityLevelState();
            state.AttemptCount++;
            if (state.AttemptCount >= LockoutConfig.MaxAttempts)
            {
                state.LockedUntilUtc = DateTime.UtcNow + LockoutConfig.LockoutDuration;
                state.AttemptCount = 0;  // reset counter, lockout takes effect
            }
            _levels[level] = state;
        }
    }

    /// <summary>
    /// v1.3.0 MINOR Item 1: clear lockout and attempt counter for a level.
    /// Called by <c>UdsClient.SecurityAccessAsync</c> on successful auth.
    /// Lockout is independent of session state — does NOT clear on session
    /// change (D8 from spec).
    /// </summary>
    public void ResetLockout(byte level)
    {
        lock (_levels)
        {
            if (_levels.TryGetValue(level, out var state))
            {
                state.AttemptCount = 0;
                state.LockedUntilUtc = null;
            }
        }
    }

    private sealed class SecurityLevelState
    {
        public byte[]? Seed { get; init; }
        public bool IsAuthenticated { get; set; }

        // v1.3.0 MINOR Item 1: lockout state
        public int AttemptCount { get; set; }
        public DateTime? LockedUntilUtc { get; set; }
    }
}
```

**Wait — the last test reveals a spec inconsistency**. D8 says "session change does NOT reset lockout", but `UdsSecurity.Reset()` is the session-change method and clears the entire `_levels` dict. The dict contains BOTH auth state AND lockout state. So `Reset()` currently clears lockout indirectly.

The correct behavior per D8 is: `Reset()` should clear auth state but NOT lockout state. Need to fix `Reset()`:

Update `Reset()` in `UdsSecurity.cs`:

```csharp
    /// <summary>
    /// Clear authentication for all levels (e.g., on session change).
    /// Lockout state is INDEPENDENT of session state — it is preserved
    /// across <see cref="Reset"/> per spec D8.
    /// </summary>
    public void Reset()
    {
        lock (_levels)
        {
            foreach (var state in _levels.Values)
            {
                state.Seed = null;
                state.IsAuthenticated = false;
            }
            // Note: AttemptCount and LockedUntilUtc are intentionally preserved.
            // Security lockout is a policy that should not be bypassed by
            // session transitions (an attacker could otherwise reset their
            // counter by triggering a diagnostic session change).
        }
    }
```

```csharp
namespace PeakCan.Host.Core.Uds;

/// <summary>
/// v1.3.0 MINOR Item 1: thrown by <c>UdsClient.SecurityAccessAsync</c> when
/// the requested security level is currently locked out (host-side
/// enforcement, before wire emit). Lockout is a per-level security policy
/// — see <see cref="UdsSecurity"/>.
/// </summary>
public sealed class UdsSecurityLockedException : UdsException
{
    /// <summary>Security level that is locked.</summary>
    public byte SecurityLevel { get; }

    /// <summary>Time remaining on the lockout window.</summary>
    public TimeSpan RemainingDelay { get; }

    public UdsSecurityLockedException(byte level, TimeSpan remaining)
        : base($"Security level 0x{level:X2} is locked; retry after {remaining.TotalSeconds:F1}s")
    {
        SecurityLevel = level;
        RemainingDelay = remaining;
    }
}
```

```csharp
        // Lockout state is independent of session state — even if a session
        // reset was attempted, the lockout window persists (D8).
        if (Security.IsLocked(level))

        byte[] requestData;

            subFunction = level;
            requestData = [subFunction];
        }
        else
        {
            // SendKey
            subFunction = (byte)(level + 1);
            requestData = new byte[1 + key.Length];
            requestData[0] = subFunction;
            Array.Copy(key, 0, requestData, 1, key.Length);

                Security.SetSeed(level, response[1..]);

                Security.SetAuthenticated(level);
                Security.ResetLockout(level);  // v1.3.0 MINOR Item 1: clear on successful auth

            when (nrc.Nrc == 0x35 || nrc.Nrc == 0x36 || nrc.Nrc == 0x37)
```

## Process lessons (from v1.2.12 + v1.2.13 + v1.2.14 PATCHes, apply to v1.3.0)

1. **Phase 2.5 actual-code exploration is mandatory.** v1.3.0 spec already absorbed it (caught "EcuReset/RoutineControl already exist"). Maintain discipline for any new items.

2. **`git diff -w main...HEAD` is the canonical CRLF check** — not applicable to v1.3.0 (no CRLF normalize this release).

3. **TDD discipline: RED first.** For Tasks 2-5: write the failing test, confirm it fails for the *right reason*, then apply the production fix, then confirm GREEN. For Task 1 (state unit tests) and Tasks 3-4 (test-coverage gap fill), the test may pass on first run if the existing code already implements the behavior — this is acceptable as "characterization testing".

4. **Self-review caught a real spec inconsistency in Task 1** — the original `DifferentLevels_Have_Independent_Counters` test asserted that `Reset()` clears lockout, contradicting D8. Fix: make `Reset()` preserve lockout state, change `SecurityLevelState.Seed` to `{ get; set; }` for in-place mutation. **This is exactly why self-review matters** — the spec was ambiguous and a careful re-read caught it.

5. **Per-level lockout is a security invariant.** D4 + D8 together mean: each security level has its own counter, lockout state is independent of session, only successful auth resets it. Any future change that wants to "auto-clear on session change" must explicitly justify breaking this invariant.

6. **Enum overloads + byte overloads pattern (D3).** Standard sub-functions get compile-time safety via enum. OEM-specific 0x40-0x7F range stays reachable via `byte` overload. Future UDS services should follow the same pattern (e.g., when adding ReadDtcInformation sub-functions).

7. **Pre-ship code-review is the last defense.** v1.2.13 caught a HIGH race that spec design missed. Don't skip.

8. **Ship method consistency.** Same as v1.2.13/14. `gh api` path is the fallback when github.com:443 blocked.

9. **Strict virtual on all UDS methods.** v1.2.14 PATCH Item 4 lifted `SendRequestAsync` and `TesterPresentAsync`. Task 3 lifts `EcuResetAsync`. After Task 3, all 9 UDS methods on `UdsClient` should be `virtual`. Verify in pre-ship review.
